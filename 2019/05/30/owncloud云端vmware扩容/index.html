owncloud云端vmware扩容

阅读数：次 2019-05-30
字数统计: 703字   |   阅读时长≈ 3分

虚拟机版的owncloud（比较麻烦，apt的源需要修改，还有证书要更新）
安装相对简单，够用，只要会修改路径就可以使用大容量内网云盘了。
自备硬盘。。。。。。

文件准备
链接：https://pan.baidu.com/s/1ejqY4IvBQXCtc-toVMpPoA
提取码：e7jp

信息准备
cmd下查询网关 ipconfig

路径查找
/var/lib/univention-appcenter/apps/owncloud/

系统环境win10 x64（64位），win7自测

  1. 安装VMware软件，操作省
  2. 导入虚拟机
  3. 安装，申请账号

修改内存步骤：
在VMware里面扩展磁盘大小
在虚拟机设置界面中选中需要扩容的磁盘, 然后在右下方处点击扩展按钮, 即可启动扩展任务:

(1) 查看已有分区
sudo fdisk -l
(2) 分区过程 输入下面命令进入fdisk命令模式(假设要分区的硬盘为/dev/sda):
sudo fdisk /dev/sda1

根据上图提示, 输入m可以显示帮助信息:
根据上图提示, 输入n添加一个新分区.
在后面的分区类型选择中输入p(创建主分区), 其余的设施保持默认值即可.
最后输入w保持分区结果.
3 格式化分区
假设通过上述步骤所创建的新分区为/dev/sda4,
可以通过下面命令将其格式化为ext4文件系统格式:
sudo mkfs -t ext4 /dev/sda44 

   4.挂载分区

1 手动挂载sudo mount /dev/sda4 /mnt/space
2 自动挂载 若希望开机时自动挂载否则重启之后分区消失找不到了, 可在
/etc/fstab最后一行添加:/dev/sda4 /mnt/space ext4 defaults 0 0
说明: 假设新分区为/dev/sda4, 挂载点为/dev/sda4 /mnt/space, 可根据实际情况修改.

进入小黑窗修改配置文件/var/lib/univention-appcenter/apps/owncloud/conf/config.php
修改第22行配置路径文件

'datadirectory' => '/var/lib/univention-appcenter/apps/owncloud/data/files',

变为

'datadirectory' => '/mnt/space/owncloud/data/files',

并且把已经储存的文件移动到该处（如果有的话）
mv /var/lib/univention-appcenter/apps/owncloud/data/files/Administrator/ /mnt/space/owncloud/data/files/

查看原目录的所有者权限
root@ucs-3887:/mnt/space/owncloud/data# ls -l /var/lib/univention-appcenter/apps/owncloud/data/
总用量 20
drwxr-xr-x 2 www-data root 4096 4月 9 00:01 custom
drwxrwx—- 5 www-data root 4096 5月 29 10:44 files
drwxr-xr-x 2 www-data root 12288 5月 29 08:09 sessions

查看当前用户组
root@ucs-3887:/mnt/space/owncloud/data# groups www-data
www-data : www-data
赋予原目录管理权限
root@ucs-3887:/mnt/space/owncloud/data# chown -R www-data:www-data /mnt/space/owncloud/data/
root@ucs-3887:/mnt/space/owncloud/data# chmod 755 /mnt/space/owncloud/data/

发现该版本没有配置网卡文件，自己创建，配置mac地址参考命令ifconfig里面，其余内容网上查询。

mkdir /etc/sysconfig/network-scripts/ -p
touch /etc/sysconfig/network-scripts/ifcfg-eth0
