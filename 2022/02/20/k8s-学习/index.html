安装kubernetes

基于单物理主机的集群

安装2022.02.20

Pod原理

Pod: kubernetes中最小单元是Pod

Pod托管了一个或多个容器；

容器<Pod<VM

Service Mesh: 多容器Pod的一个”以基础设施为中心”

proxy container：每个Pod都含有一个代理容器，类似路由器管理流量

微服务

分离需求点，一个容器实现一个需求点

运行Pod

Pod只是运行应用的载具，运行或部署Pod = 运行部署应用

Pod与共享网络

每个Pod都含有一个IP地址、一组TCP和UDP端口范围，以及一个路由表。多容器的Pod也是一样。

Pod内部容器通过localhost访问外部通过ip：port访问

不止共享网络以及卷、内存、IPC套接字等资源。换而言之，Pod持有的命名空间被Pod中的容器继承和共享。

Pod与CGroup

用于限制资源的使用

一个Pod中的容器可以含有不同的CGroup资源限制。

Pod的原子部署

部署一个Pod是一种原子操作。要么成功要么失败。成功意味着Pod中所有容器部署到同一节点上。

Pod的生命周期

  1. 定义yaml，传送API server
  2. 等待状态，pending state
  3. 运行状态，runing state

Pod挂了之后，会使用另一个进行替代，同样的配置有不同的ID与IP，Pod不依赖IP

Pod清单

例子

apiVersion: :， api-group可以忽略，eg：storage.k8s.io/v1 ; v1 都是正确的

kind：告诉kubernetes要部署的类型，这里是Pod对象

metadata：定义名称和标签。namespace可以用于从管理层面对集群进行逻辑上的划分。并不能做安全屏障来使用。这里使用default作为命名空间

spec：定义pod所运行的容器。基于镜像nigelpoulton/k8sbook:1.0命名为hello-ctr并暴露8080端口

如果是多容器可以在spec部分增加其他容器的定义

基于Pod清单部署

查看运行中的Pod输出内容分为两部分spec和status(期望状态和当前状态)

kubectl describe

监控状态

kubectl exec

在pod中执行命令

kubectl logs 查看日志
